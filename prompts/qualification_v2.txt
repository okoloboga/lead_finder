Контекст: Анализируем потенциального клиента для пользователя сервиса Lead Finder.

## Услуги пользователя

{services_description}

и контекст реально соотносятся с услугами пользователя.

## Входные данные

1. Профиль пользователя в Telegram
2. Его личный канал (если есть)
3. Результаты веб-поиска (если есть)
4. Его сообщения в тематическом чате
5. Актуальные AI-решения для его ниши (если есть)

## Задача

### 1. ИДЕНТИФИКАЦИЯ
- Владелец бизнеса или наёмный/блогер/новичок?
- Какой бизнес, ниша?
- Масштаб (микро/малый/средний)?

### 2. КВАЛИФИКАЦИЯ (0-5)

| Score | Критерии |
|-------|----------|
| 5 | Владелец + ПРЯМАЯ боль из сообщений + AI решает её чётко |
| 4 | Владелец + явная боль + AI-решение реалистично |
| 3 | Владелец + есть боль + AI-решение возможно |
| 2 | Владелец + слабая боль или решение неочевидно |
| 1 | Владелец + мало данных, но AI-потенциал виден |
| 0 | НЕ подходит |

**Score = 0 если:**
- НЕ владелец бизнеса
- НЕТ конкретных болей из данных (только предположения)
- Боль НЕ решается AI-инструментами

**ВАЖНО:** Не придумывай "типичные боли" — только из реальных данных!

### 3. ВЫЯВЛЕННЫЕ БОЛИ

Указывай ТОЛЬКО то, что ЯВНО видно:
- Цитаты из сообщений в чатах
- Жалобы в постах канала
- Прямые запросы на решение

❌ Не придумывай боли!

### 4. ИДЕЯ AI-РЕШЕНИЯ

Предложи конкретное AI-решение под боль. Используй актуальные AI-решения для ниши (если переданы).

Примеры маппинга боль → AI-решение:

| Боль | AI-решение |
|------|------------|
| Много однотипных вопросов | AI-ассистент / RAG на базе знаний |
| Теряем заявки, менеджеры не успевают | AI-агент приёма и квалификации заявок |
| Рутинные задачи съедают время | AI-автоматизация процесса |
| Нет понимания что происходит в бизнесе | AI-аналитика, авто-отчёты с инсайтами |
| Ручная обработка документов/данных | AI-парсинг, классификация, экстракция |
| Клиенты не получают быстрый ответ | AI-ассистент 24/7 в любом канале |
| Сложно масштабировать поддержку | AI-первая линия поддержки |
| Нет персонализации в маркетинге | AI-сегментация и персонализированный контент |

Не ограничивай себя этим списком — предлагай то, что реально решает боль.

### 5. АУТРИЧ (только если score >= 1)

- Hook: за что цепляемся (конкретное сообщение или пост)
- Короткое персонализированное сообщение (3-5 предложений)
- Упомяни конкретный AI-инструмент под их задачу
- Без агрессивных продаж

Если score = 0 → outreach = null

## Формат ответа

**Лид подходит (score 1-5):**
```json
{
  "identification": {
    "is_business_owner": true,
    "business_type": "Селлер Wildberries, детская одежда",
    "business_scale": "Малый бизнес"
  },
  "qualification": {
    "score": 4,
    "reasoning": "Владелец, прямая боль про отзывы, AI чётко решает"
  },
  "identified_pains": [
    "Тратит 3 часа в день на ответы на отзывы WB"
  ],
  "product_idea": {
    "idea": "AI-агент для генерации и публикации ответов на отзывы",
    "pain_addressed": "Сокращение времени с 3 часов до 15 минут",
    "estimated_value": "15+ часов в неделю"
  },
  "outreach": {
    "hook": "Сообщение про 3 часа на отзывы",
    "message": "Привет! Видел про отзывы — 3 часа в день это много. Мы делаем AI-агентов, которые сами генерируют и отправляют ответы под стиль бренда. Хочешь покажу как это работает?"
  }
}
```

**Лид НЕ подходит (score = 0):**
```json
{
  "identification": {
    "is_business_owner": false,
    "business_type": "Не определён",
    "business_scale": "Не применимо"
  },
  "qualification": {
    "score": 0,
    "reasoning": "Не владелец / нет конкретных болей / боль не решается AI"
  },
  "identified_pains": [],
  "product_idea": null,
  "outreach": null
}
```
