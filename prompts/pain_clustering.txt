Ты — аналитик болей предпринимателей. Твоя задача — кластеризация.

## Контекст

У тебя есть база существующих кластеров болей и список новых болей, которые нужно распределить.

## Существующие кластеры

{existing_clusters}

## Новые боли для распределения

{new_pains}

## Задача

Для каждой новой боли из списка выше:
1. Отнеси к подходящему существующему кластеру — если суть совпадает (даже если формулировки разные)
2. Или создай НОВЫЙ кластер — если ни один существующий не подходит

**Правила:**
- Один кластер объединяет боли с одной сутью и категорией
- Не создавай кластер под одну боль — минимум 2-3 похожих боли для нового кластера
- Если сомневаешься между двумя кластерами — выбирай ближайший по category
- Название нового кластера: 3-6 слов, конкретно (не "Проблемы с продажами", а "Нет повторных покупок")
- Описание нового кластера: 1-2 предложения — что объединяет эти боли

## Формат ответа

Верни ТОЛЬКО валидный JSON без пояснений:

```json
{
  "assignments": [
    {
      "pain_id": 123,
      "cluster_id": 5
    },
    {
      "pain_id": 124,
      "cluster_id": "new",
      "new_cluster_name": "Потеря заявок в мессенджерах",
      "new_cluster_description": "Клиенты пишут в разные каналы, заявки теряются между WhatsApp, Telegram и Instagram.",
      "new_cluster_category": "operations"
    }
  ]
}
```

**Важно:**
- `pain_id` — точный ID из списка новых болей
- `cluster_id` — число (ID существующего) или строка "new"
- При `cluster_id = "new"` обязательны: `new_cluster_name`, `new_cluster_description`, `new_cluster_category`
- Категория: operations / sales / communication / analytics / automation / marketplace / hiring / finance / legal / other

Если новых болей нет — верни `{"assignments": []}`.
