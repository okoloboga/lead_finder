# LeadCore SaaS — Техническое задание

## Цель

Превратить текущего бота (работает только для одного пользователя) в SaaS-продукт с мультитенантностью, подписками и админкой.

---

## Текущее состояние

Бот умеет:
- Создавать программы поиска (ниша + чаты)
- Парсить участников чатов
- Квалифицировать лидов через LLM
- Сохранять ссылки на сообщения и профили
- Кластеризовать боли
- Генерировать контент на основе болей

**Проблема:** Всё завязано на одного пользователя. Нет разделения данных, нет оплаты.

---

## Новый функционал

### 1. Мультитенантность (разные пользователи)

**Что нужно:**
- Каждый пользователь видит только свои данные
- Отдельные программы, лиды, кластеры для каждого
- Изоляция данных между пользователями

**Изменения в БД:**

Добавить таблицу `users`:
```
users:
- telegram_id (primary key)
- username
- services_description (текст — чем занимается)
- subscription_type (free / paid)
- subscription_expires_at (datetime)
- created_at
- last_active_at
```

Добавить `user_id` во все существующие таблицы:
- programs → добавить user_id
- leads → добавить user_id
- clusters → добавить user_id
- content_drafts → добавить user_id

**Логика:**
- При любом запросе фильтровать по user_id
- Новый пользователь → создаётся запись в users → проходит онбординг

### 2. Онбординг

**Триггер:** Первое сообщение от нового пользователя (user_id не найден в БД)

**Флоу:**

```
Бот: Привет! Я помогаю находить клиентов в Telegram-чатах.

Чтобы настроить поиск под тебя — расскажи в одном сообщении:
• Какие услуги ты продаёшь?
• Кто твои клиенты?

Например: "Делаю сайты и лендинги для малого бизнеса" 
или "Настраиваю рекламу для интернет-магазинов"

Пользователь: [вводит текст]

Бот: Отлично! Сохранил. Теперь я буду искать лидов, 
которым нужны именно твои услуги.

[Показывает главное меню]
```

**Сохранение:**
- Текст пользователя → `users.services_description`
- Этот текст используется в промпте квалификации вместо захардкоженного описания услуг

```
Услуги пользователя:

```

**Логика:**
- При квалификации лида подставлять `services_description` из профиля пользователя
- Остальная структура промпта остаётся

### 4. Система подписок

**Тарифы:**

| Тип | Ограничения | Цена |
|-----|-------------|------|
| Free | 1 программа, 1 анализ в неделю | 0 |
| Paid | Без ограничений | Telegram Stars |

**Периоды подписки (Paid):**
- 1 месяц
- 3 месяца
- 6 месяцев
- 12 месяцев

**Цены в Telegram Stars (XTR):**
- 1 месяц — ⭐ 500
- 3 месяца — ⭐ 1200
- 6 месяцев — ⭐ 2000
- 12 месяцев — ⭐ 3500

**Логика проверки лимитов (Free):**

При создании программы:
```python
if user.subscription_type == 'free':
    program_count = count_programs(user_id)
    if program_count >= 1:
        return "На бесплатном тарифе доступна только 1 программа. 
                Оформи подписку для снятия лимита."
```

При запуске анализа:
```python
if user.subscription_type == 'free':
    last_analysis = get_last_analysis_date(user_id)
    if last_analysis and (now - last_analysis) < 7 days:
        days_left = 7 - (now - last_analysis).days
        return f"На бесплатном тарифе 1 анализ в неделю. 
                Следующий доступен через {days_left} дн."
```

### 5. Оплата через Telegram Stars

**Флоу оплаты:**

```
Пользователь: [нажимает "Подписка"]

Бот: 💎 Подписка LeadCore

Бесплатно: 1 программа, 1 анализ в неделю
Платно: без ограничений

Выбери период:
[1 мес — ⭐ 500]
[3 мес — ⭐ 1200]
[6 мес — ⭐ 2000]
[12 мес — ⭐ 3500]

Пользователь: [выбирает период]

Бот: [отправляет invoice через Telegram Payments API]

Пользователь: [оплачивает звёздами]

Бот: ✅ Подписка активирована до [дата]!
```

**Техническая реализация:**

Использовать `sendInvoice` с `currency: "XTR"` (Telegram Stars):

```python
await bot.send_invoice(
    chat_id=user_id,
    title="LeadCore — подписка 1 мес",
    description="Безлимитный поиск лидов",
    payload=f"subscription_{user_id}_1month",
    currency="XTR",
    prices=[{"label": "Подписка 1 мес", "amount": STARS_PRICES["1m"]}]
)
```

Обработать `pre_checkout_query` и `successful_payment`:

```python
@bot.on_message(filters.successful_payment)
async def on_payment(message):
    user_id = message.from_user.id
    payload = message.successful_payment.invoice_payload
    period = parse_period(payload)  # 1month, 3months, etc.
    
    update_subscription(user_id, period)
```

**После оплаты:**
- `subscription_type` = 'paid'
- `subscription_expires_at` = now + период

**Проверка истечения:**
- Ежедневный cron проверяет `subscription_expires_at`
- Если истекла → `subscription_type` = 'free'
- Опционально: уведомление за 3 дня до истечения

## План реализации

### Текущий статус (факт)
- Реализация идёт в режиме **чистой БД** (без миграций старых данных).
- Выполнено: базовая мультитенантность по `user_id` и разрыв логики `owner_chat_id`/автосбора.
- Выполнено: базовый онбординг `services_description`, экран редактирования в настройках, динамический промпт квалификации.
- Выполнено: базовые лимиты Free (1 программа, 1 запуск в 7 дней) с проверками в создании и запуске.
- Выполнено: автозапуск/ручной запуск вынесен в фоновые задачи через `Celery + Redis` (scheduler только ставит задачу в очередь).
- Выполнено: устранена ошибка asyncpg `cannot perform operation: another operation is in progress` (процессная привязка SQLAlchemy engine + Celery `--pool=solo`).
- Частично выполнено: Telegram Stars (инвойс XTR + pre_checkout + successful_payment).
- Частично выполнено: админка (вход `/admin_panel`, сводка, поиск пользователя, ручная выдача подписки).
- Не выполнено: автоматический cron-перевод paid→free, расширенная админ-аналитика по периодам.
